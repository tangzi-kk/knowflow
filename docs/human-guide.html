KnowFlow 组件阅读说明

本项目由两个独立运行组件组成：Obsidian 插件负责本地知识库读写与同步服务，浏览器扩展负责在飞书网页侧提供入口和界面。两者配合使用，但安装位置、文件结构和验证方式不同。

一眼区分

分类 组件名 最新版本 仓库内位置 安装/运行位置
Obsidian 插件 fs-TB 3.2.1 releases/obsidian-fs-TB/3.2.1/ 目标 vault 的 .obsidian/plugins/fs-TB/
浏览器扩展 Feishu Doc Exporter 0.3.0 releases/browser-feishu-doc-exporter/0.3.0/ Chrome/Edge 扩展管理页加载已解压目录
不要把浏览器扩展目录放进 Obsidian，也不要把 Obsidian 插件文件加载到 Chrome/Edge。两个组件通过本地服务和同步令牌配合。

组件职责

Obsidian 插件

fs-TB

运行在 Obsidian 内，负责监听本地 HTTP 服务、调用 lark-cli、写入 vault 文件、处理 YAML frontmatter、图片 token、callout 转换和回写。

运行产物通常是 main.js、manifest.json、styles.css。

浏览器扩展

Feishu Doc Exporter

运行在 Chrome/Edge 中，负责在飞书网页注入同步入口、侧边栏、弹窗和配置界面，将当前飞书文档信息发送给 Obsidian 插件。

运行产物通常包含 manifest.json、background.js、content.js、icons/。

开发目录

路径 用途
packages/ob-plugin/ Obsidian 插件源码与构建入口。
extension/ 浏览器扩展源码与构建入口。
packages/shared/ 两个组件共用的协议、YAML、callout、hash、图片和文件名处理逻辑。
releases/ 已核验运行产物归档，按 Obsidian 插件和浏览器扩展分目录保存。

常用验证命令

npm run test:protocol
npm run build
git diff --check
