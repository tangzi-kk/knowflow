# KnowFlow 4.0 验收边界

## 自动验收

统一命令：

```bash
npm run release:verify
git diff --check
```

覆盖：版本一致性、设置迁移、协议错配、唯一绑定、20 路幂等、并发锁、路径攻击、三方冲突、恢复副本、浏览器秘密存储、操作重启、迟到流消息、最小权限、确认式删除、活动损坏、HTTP 鉴权/方法/体积/超时、CLI 重试边界和构建产物 SHA256。

每个失败路径至少断言：不静默覆盖、不报告假成功、保留明确错误或恢复线索。

## 必须真机完成，自动测试不能代替

- 使用真实飞书账号执行 Pull、Pushback、429、登录过期和图片下载。
- 在备份 Vault 中验证 3.2.1 → 4.0.0 升级、4.0.0 → 3.2.1 回滚和插件卸载。
- 在 Chrome/Edge 开发者模式加载 `extension/dist/`，人工终止 MV3 Service Worker 后检查操作终态。
- 验证端口占用、Obsidian 未启动、错误 Token 和浏览器权限拒绝时的界面提示。

没有真实账号、Token、目标 Vault 和浏览器交互授权时，这些项目只能标记为“未执行”，不能写成通过。
